att.linguistic

att.linguistic provides a set of attributes concerning linguistic features of tokens, for usage within the element w in the analysis module.
Moduleanalysis — Simple Analytic Mechanisms
Membersw
AttributesAttributes
lemma lemma¶provides a lemma (base form) for the word, typically uninflected and serving both as an identifier (e.g. in dictionary contexts, as a headword), and as a basis for potential inflections.
Status Optional
Datatype teidata.text
<w lemma="year">yeare</w>
<w lemma="Arznei">Artzeneyen</w>
lemmaRef lemmaRef¶provides a pointer to a definition of the lemma for the word, for example in an online lexicon.
Status Optional
Datatype teidata.pointer
<w type="verb" lemma="hit"
 lemmaRef="http://www.example.com/lexicon/hitvb.xml">hitt<m type="suffix">ing</m>
</w>
pos pos¶(part of speech) indicates the part of speech assigned to a token (i.e. information on whether it is a noun, adjective, or verb), usually according to some official reference vocabulary (e.g. for German: Stuttgart-Tuebingen-Tagset [STTS], for English: CLAWS, for Polish: NKJP, etc.).
Status Optional
Datatype teidata.text
<w pos="vvz">is</w>
<w msd="vvg">taking</w>
<w pos="NN">Universitäten</w>
<w pos="VVFIN">fragte</w>
msd msd¶(morphosyntactic description) supplies morphosyntactic information for a token, usually according to some official reference vocabulary (e.g. for German: Stuttgart-Tuebingen-Tagset [STTS] -- large tag set; for English XXX).
Status Optional
Datatype teidata.text
<ab>
 <w pos="PPER" msd="1.Pl.*.Nom">Wir</w>
 <w pos="VVFIN" msd="1.Pl.Pres.Ind">fahren</w>
 <w pos="APPR" msd="Akk">in</w>
 <w pos="ART" msd="Def.Masc.Akk.Sg.">den</w>
 <w pos="NN" msd="Masc.Akk.Sg.">Urlaub</w>
</ab>
reg reg¶(regular form) provides the normalized/standardized form if a token is written in non-standard orthography.
Status Optional
Datatype teidata.text
<w reg="Teil">Theyll</w>
<w reg="Freude">Frewde</w>
join join¶provides information on the absence of whitespace between a token and its preceding/following token.
Status Optional
Datatype teidata.text
Legal values are:
left
(there is no whitespace on the left side of the token)
right
(there is no whitespace on the right side of the token)
both
(there is no whitespace on either side of the token)
